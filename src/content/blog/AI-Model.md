# Claude Sonnet 5 — 值得注意的改變與模型選擇指南


模型發布日期：2026/06/30


---


## 四個值得關注的改變


**自主執行多步驟任務**
能自己規劃步驟、呼叫瀏覽器和終端機、中途遇到問題會調整方向。以前只能用Opus做此操作。


**知識工作分數超過 Opus 4.8**
GDPval-AA v2 這個 benchmark 測的是真實文件處理任務。Sonnet 5 在這裡略勝 Opus 4.8（1,618 vs 1,615）—— 差距很小，但文件類工作不需要用到旗艦模型處理。


**比 4.6 少幻覺**
幻覺率和 sycophancy（附和使用者說錯的事）都比 Sonnet 4.6 低。跑 agent 流程時，這個差異比跑單次對話時更明顯。


**Adaptive Thinking 預設開啟**
模型自己判斷這個任務需不需要深度推理，不用手動設定。可以用 low / medium / high / max 控制，依據個人使用情境來控制思考速度和cost。

## 哪種任務用哪個模型


### 🔵 Sonnet 5 — 文件與知識整理

- 需求分析
- 技術規格撰寫
- 設計決策記錄（ADR）
- Code Review 彙整
- 會議紀錄與追蹤

### 🔵 Sonnet 5 — 自動化流程

- 多步驟 agent 任務
- 瀏覽器操作
- 資料蒐集與整理
- 中等複雜度程式任務
- CI/CD 腳本輔助

### 🟣 Opus 4.8 — 精準度要求高的開發

- 複雜架構設計
- DDD Aggregate 邊界設計
- 大型重構
- 深度 debug
- 高風險邏輯推理

### 🟢 Haiku 4.5 — 高頻輕量任務

- 格式轉換
- 分類與標記
- 固定樣板產生
- 大量 API 呼叫
- 快速補全

## 調整使用習慣的出發點


Sonnet 5 推出後，提供了更經濟實惠的模型，文件、規格、agentic 流程交給 Sonnet 5處理。


程式碼生成和高風險推理才動用 Opus 4.8。


高頻、格式固定的任務用 Haiku。


明確規範不同任務狀態，選用適合模型，既可以省Token也可以準確地完成開發任務。除了Ponytail的Skill等，模型選用調教也是一個很重要的設定。
